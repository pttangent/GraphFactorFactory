## 结论

1. 日内连续性显著存在。此前 2026-01-22 的一分钟真实 continuation 约为 60% 至 64%，明显高于 shifted-time 与 member-permutation null。
2. 相同阈值不能直接用于 overnight。Jaccard 0.35 与 hybrid-strict 在两个跨夜边界均没有任何匹配。
3. overlap containment 0.50 能产生少量跨夜匹配，但平均 retention 只有 0.923%，且不高于 day-order null，因此目前不能把这些匹配解释为可靠的跨夜主题继承。
4. 跨夜匹配的平均 Jaccard 仅约 0.045，说明这些事件多为“小社区被更大社区包含”，而不是成员结构稳定延续。
5. 当前证据支持“日内主题连续、隔夜大幅重置”，不支持使用同一社区 identity 无条件跨日继承。

## 建议实现

- 日内 lifecycle 保持现有 effective-state tracker。
- 收盘后将路径置为 dormant，而不是直接继续 active age。
- 次日开盘使用独立 overnight matcher，并要求：
  - 至少两个独立结构证据；
  - Jaccard 与 containment 同时设低门槛，而非只使用 containment；
  - 对社区规模膨胀施加惩罚；
  - 将 unmatched 开盘社区视为新生主题。
- revival 必须单独标记，不应与日内 continuation 混合统计。

## 限制

容器重置导致此前完整三日 path checkpoint 丢失。本次 overnight 结果来自重新计算的六个边界状态，足以回答跨夜结构是否稳定，但不能恢复原本全部 559 states 的逐路径 split/merge/death/revival 明细。
